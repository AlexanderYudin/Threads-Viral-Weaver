# Threads Viral Weaver

## Обзор проекта
- **Название**: Threads Viral Weaver
- **Цель**: Поиск «залётных» (виральных) постов в Threads по ключевым словам на заданную тему (например «отношения, знакомства, дейтинг»), расчёт ER (engagement rate) и виральности, отбор референсов для рерайта и публикации.
- **Контекст**: Сделано по задаче — мониторить не блогеров, а все публикации по ключам за период (24ч/7д/30д), считать ER и находить вирусные посты для переписывания.

## Что уже реализовано
- ✅ Поиск по списку ключевых слов (через запятую/с новой строки) + пресеты тем
- ✅ Фильтры: период (24ч / 7д / 30д / всё время), тип контента (текст/фото/видео), мин. viral score
- ✅ Сортировка: по виральности / по ER / по лайкам / по свежести
- ✅ Расчёт **ER** (по просмотрам или подписчикам) и **Viral Score** (абсолютная вовлечённость + скорость набора + ER)
- ✅ Карточки постов: автор, текст, метрики (лайки/ответы/репосты/просмотры), бейдж «залётности»
- ✅ Модалка с полным текстом поста + кнопка «Скопировать» (для рерайта) и ссылка на оригинал
- ✅ Сводка: найдено постов, макс. viral score, средний ER, суммарная вовлечённость
- ✅ 4 провайдера данных (переключаются в «Источник данных»):
  - 🧪 **Demo** — реалистичные данные дейтинг-тематики, работает без ключей (для теста UI и логики)
  - 🔵 **Official Threads API** — `graph.threads.net/keyword_search`, нужен access token. Возвращает текст/автора/время, но НЕ отдаёт метрики чужих постов → ER недоступен
  - 🟢 **ScrapeCreators** — неофициальный, нужен x-api-key, возвращает полные метрики → ER считается
  - 🟣 **EnsembleData** — неофициальный, нужен token, возвращает полные метрики → ER считается

## Функциональные точки входа (API)
- `GET /` — веб-интерфейс
- `POST /api/search` — поиск постов
  - Тело (JSON):
    ```json
    {
      "keywords": ["отношения", "dating"],
      "provider": "demo|official|scrapecreators|ensembledata",
      "searchType": "TOP|RECENT",
      "period": "24h|7d|30d|all",
      "mediaType": "ALL|TEXT|IMAGE|VIDEO",
      "minViral": 0,
      "sortBy": "viral|er|likes|recent",
      "limit": 30,
      "token": "<ключ провайдера, опционально>"
    }
    ```
  - Ответ: `{ provider, query, total, metricsAvailable, posts[], notes[] }`

## Модель данных
- **ThreadsPost**: id, text, username, displayName, avatarUrl, permalink, timestamp, mediaType, likes, replies, reposts, quotes, views, followers + вычисляемые engagement, er, viralScore, ageHours
- **Хранилище**: нет (приложение stateless). Токены провайдеров хранятся только в браузере (localStorage), на сервер не сохраняются.
- **Поток данных**: фронтенд → `/api/search` → выбранный провайдер (внешний API или генератор demo) → обогащение метриками (scoring.ts) → фильтры/сортировка → JSON → рендер карточек.

## Логика расчёта (scoring.ts)
- **ER** = вовлечённость / просмотры × 100 (или / подписчиков, если просмотров нет)
- **Viral Score** = log10(вовлечённость) × 12 + скорость набора (вовлечённость/час, логарифмически, до 30) + ER-компонент (до 20)
- Бейджи: ≥70 «🔥 мощно залетел», ≥50 «📈 залетает», ≥30 «👀 неплохо»

## Руководство пользователя
1. Введите ключевые слова (или выберите пресет) — тема «отношения/знакомства/дейтинг» уже подставлена.
2. Выберите период, сортировку, тип контента, лимит.
3. (Опционально) Нажмите «Источник данных» → выберите провайдера и вставьте ключ. По умолчанию — Demo (без ключа).
4. Нажмите «Найти залётные посты». Посты отсортируются по виральности.
5. Откройте карточку → «Текст / рерайт» → скопируйте текст для переписывания, либо откройте оригинал.

## Как получить реальные метрики
- Официальный Threads API не отдаёт лайки/просмотры чужих постов. Для подсчёта ER используйте:
  - **ScrapeCreators** (scrapecreators.com) — есть keyword search с метриками
  - **EnsembleData** (ensembledata.com) — Threads keyword search с метриками
  - **Apify** — Threads Search Scraper (требует Session ID)
